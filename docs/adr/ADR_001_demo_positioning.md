# ADR 001 — Demo/Product Prototype Positioning

## Status
Accepted

## Context

The repository is intended as a portfolio-grade CRM and AI automation case. It must look production-capable without misrepresenting commercial deployment status.

## Decision

- Position the repository as a demo/product prototype.
- Keep architecture production-capable, but implementation demo-first.
- Use synthetic data only in public materials.
- Never claim automatic client messaging or production rollout.

## Consequences

- The repo stays honest and publishable.
- Live validation can exist without changing the public positioning.

## Alternatives Considered

- Presenting the repo as commercial production software.
- Removing all real-ready boundaries to stay purely mock-only.

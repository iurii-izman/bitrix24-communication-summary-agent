# Support

## Before Opening An Issue

- Read [README.md](./README.md) for project scope and setup.
- Read [CONTRIBUTING.md](./CONTRIBUTING.md) before proposing changes.
- Check [docs/project_status.md](./docs/project_status.md) for current boundaries.
- For validation or cleanup questions, read [docs/bitrix_cleanup_runbook.md](./docs/bitrix_cleanup_runbook.md).

## Use The Right Channel

- Bug report:
  Open a GitHub issue with the bug form and include reproduction steps.
- Feature request:
  Open a GitHub issue with the feature request form and explain the user-facing value.
- Security concern:
  Follow [SECURITY.md](./SECURITY.md). Do not post secrets, tokens, or sensitive payloads publicly.
- General repo question:
  Use GitHub Discussions if enabled, otherwise open an issue labeled `question`.

## What Helps

- exact command used
- expected behavior
- actual behavior
- logs or screenshots with secrets and PII removed
- whether you were in demo mode, dry-run real mode, or write-enabled validation mode

## What Does Not Help

- screenshots containing `.env` values
- raw webhook URLs
- real customer data
- vague reports without reproduction steps

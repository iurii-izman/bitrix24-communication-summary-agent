# Code Of Conduct

## Our Standard

This repository is maintained as a pragmatic, respectful engineering space.

Expected behavior:
- discuss code and behavior, not people
- assume good intent until evidence says otherwise
- keep bug reports reproducible and specific
- avoid posting secrets, real customer data, or unmasked PII
- use synthetic examples for public discussion

Unacceptable behavior:
- harassment, insults, or intimidation
- doxxing or sharing private information
- publishing credentials, webhook URLs, or production identifiers
- disruptive spam or bad-faith issue/PR churn

## Scope

This code of conduct applies to repository issues, pull requests, discussions, and review comments.

## Enforcement

Project maintainers may edit, hide, lock, or remove content that violates these rules, and may restrict further participation when necessary.

## Reporting

For conduct or moderation concerns, use the support path described in [SUPPORT.md](./SUPPORT.md). Do not open a public issue for sensitive incidents.

## Notes

This file is intentionally short and repository-specific. If the project later expands into a broader public collaboration space, replace it with a more formal policy such as Contributor Covenant.
